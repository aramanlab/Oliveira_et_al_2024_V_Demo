There are two Sections:
  A. Creating and Fitting the Random Forest Model to Strain presence-absence and KP Load
  B. Exploring a Space of 1,000,000 Communities In-Silico Using the model generated in A.

--------------------------------------------------------------------------------------------------------------------------
Section A: Creating and Fitting the Random Forest Model to Strain presence-absence and KP Load [Run time 5 minutes]
--------------------------------------------------------------------------------------------------------------------------
1. We Separate the Strain-presence absence matrix from the KP Load. (Separation of Independent and Dependent Variables for RF)
    This is done by the function - 'read_KP_data'

2. Then we split the data into Training and Test Set using the function -'split_data_train_test'.

3. We then create the RF model using the Training Strain Matrix and Training Log[KP CFU] values.
    This is done by the function -'build_RF_model'

4. Testing of the RF model is done by the function -'test_RF_model' on the Test set.

5. We then save the RF model for Section B.


__________________________________________________________________________________________________________________________
--------------------------------------------------------------------------------------------------------------------------
Section B: Exploring a Space of 1,000,000 Communities In-Silico Using the model generated in A. [Run time 40 minutes]
--------------------------------------------------------------------------------------------------------------------------
  1. Construct a one-hot-encoded community of length 46. (We have 46 total strains.)
      - This is a binary vector of length 46 with 1s representing the presence and 0 representing the absence of any strain.
      - Duplicate experiments have been removed from the community. So we are left with unique communities only.

  2. For each of the communities above we predict the Log[KP CFU] with RF model from Section 1.

  3. We then bin the Log[KP CFU] in to create a histogram of performance for the communities.
